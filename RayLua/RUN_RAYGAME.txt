YES, IT IS ALREADY COMPILED.

The built game binary is here:

RayLua/build/RayGame

What you need on this Arch Linux machine:

1. raylib installed
2. OpenGL working
3. the files already inside this repo

How to run it from the repo root:

cd /home/jaba/Documentos/raygame/duke3d-build2-main
./RayLua/build/RayGame /path/to/your.map

If you do not pass a map, or the paths are wrong, RayGame now opens a startup menu instead of closing.

Startup menu fields:

- Map (.map)
- Install Dir
- Primary Assets
- Secondary Gallery

Useful buttons:

- Use Map Dir
- Use Install Dir
- Validate
- Start RayGame

Important asset rule:

- the map directory must also contain `palette.dat`
- the map directory must also contain `TILES*.ART`
- optional extra gallery directory:
  `Content/GAL_002_SW/`

Example idea:

./RayLua/build/RayGame /path/to/duke3d/e3l3.map

If you want to rebuild it later:

cd /home/jaba/Documentos/raygame/duke3d-build2-main
cmake -S RayLua -B RayLua/build -G Ninja -DCMAKE_C_COMPILER=gcc -DCMAKE_CXX_COMPILER=g++ -DDISABLE_LUA=ON -DBUILD_DUKEGAME=OFF
cmake --build RayLua/build -j4

Important:

- This verified build is RayGame, not DukeGame.
- Lua is disabled in this Linux build.
- Shaders are copied into RayLua/build/Shaders automatically.
- script.lua is copied into RayLua/build automatically.
- The old Windows font crash was patched. RayGame now tries Linux fonts first and falls back to ImGui default font.

If you want to run it from inside the build folder instead:

cd /home/jaba/Documentos/raygame/duke3d-build2-main/RayLua/build
./RayGame

If it says a library is missing, install:

- raylib
